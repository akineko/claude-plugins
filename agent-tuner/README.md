# agent-tuner

> **非推奨**: 現在は Anthropic 公式の `/skill-creator` プラグインがアップデートされ、同等以上の品質でスキル作成・レビューが可能になっています。新規利用には `/skill-creator` を推奨します。

AI エージェントの環境構築・最適化を支援するプラグイン。

スキルやエージェントの作成、メモリ・設定の最適化など、AI コーディングツールの活用を調整・強化するツールキットを提供する。

## スキル

- `/skill-builder`
  - 対話形式でベストプラクティスに従ったスキルを作成します。
    - チャットによる作業終了後に「これをスキルにして」といった作成にも対応
  - スクリプトが必要と判断した時はプロジェクトの採用言語を優先します。
    - 他言語を採用時はユーザーに理由を説明しての承認制

- `/skill-reviewer`
  - 既存スキルをベストプラクティスに基づいてレビューし、具体的な改善レポートを生成します。
  - 重大度（❌ 動作不可 / ⚠️ 品質低下 / ✅ 良好）を明示し、インパクト順に改善点を提案します。

### 補足

スキル作成とスキルレビューのスキルは [The Complete Guide to Building Skills for Claude](https://claude.com/blog/complete-guide-to-building-skills-for-claude) のベストプラクティスをベースに作成しました。
スクリプトの採用言語の判断フロー追加やガイドにはないベストプラクティスを追加したりのカスタマイズも加えているので完全に準拠しているわけではないです。

### Anthropic 提供の `/create-skill` ではない理由

- 作られる description のトリガーワードが弱く作成後に改善が必要な事が多い
- スクリプトを採用しがちで更に言語も Python が選ばれがち
- `/create-skill` のフロー自体にスクリプトが含まれており Python 実行の許可が求められる
- 最後にZIPでパッケージングする処理を必ず実行しようとする

## 提供予定のスキル・エージェント

- **サブエージェント作成** — ベストプラクティスに従ったサブエージェントの設計・作成を支援
- **メモリ最適化** — プロジェクトのメモリ・設定ファイルを整理・最適化
