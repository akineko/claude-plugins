# カテゴリ別チェック項目

スキルがどのカテゴリに該当するかを判断し、該当するカテゴリの追加チェックを行う。

各項目の重大度:
- ❌ **要修正** — ガイドライン違反、または重大な問題がある
- ⚠️ **改善推奨** — 動作はするが改善の余地がある

---

## カテゴリの判別方法

以下の特徴からカテゴリを判断する:

| カテゴリ | 主な特徴 |
|---------|---------|
| 1: ドキュメント＆アセット生成 | ファイル出力（docx, pptx, html等）が主目的。外部ツール不要の場合が多い |
| 2: ワークフロー自動化 | 複数ステップの処理を一貫した方法論で実行。複数のMCPサーバーの連携を含む場合もある |
| 3: MCP連携強化 | 特定のMCPサーバーの利用を前提とし、そのワークフローガイダンスを提供 |

複数のカテゴリにまたがるスキルもある。その場合は該当する全てのカテゴリのチェックを行う。

---

## カテゴリ1: ドキュメント＆アセット生成

代表例: frontend-design, docx, pptx, xlsx スキル

### チェック項目

#### ❌ 要修正
- [ ] **出力形式の明示**: 生成されるファイルの形式が明確か

#### ⚠️ 改善推奨
- [ ] **スタイルガイド・ブランド基準**: 出力のデザインや書式に関する具体的な基準が定義されているか
  - フォント、色、レイアウトなどの指定
  - 文体やトーンのガイドライン
- [ ] **テンプレート構造**: 一貫した出力を生成するためのテンプレートが提供されているか
  - assets/ディレクトリにテンプレートファイルが含まれているか
  - テンプレートの使い方がSKILL.md内で説明されているか
- [ ] **品質チェックリスト**: 出力ファイナライズ前の品質確認項目があるか
  - 出力の必須要素の確認
  - フォーマットの一貫性チェック

---

## カテゴリ2: ワークフロー自動化

代表例: skill-creator スキル

### チェック項目

#### ⚠️ 改善推奨
- [ ] **バリデーションゲート**: 各ステップ間にバリデーション（確認・検証）のポイントが設けられているか
  - 次のステップに進む前に、前のステップの成果物を確認するフローがあるか
- [ ] **テンプレート提供**: よくある構造のテンプレートが用意されているか
  - ユーザーがゼロから始めなくて済む仕組み
- [ ] **レビュー・改善サジェスト**: 出力に対するレビューや改善提案の仕組みがあるか
- [ ] **反復改善ループ**: 一度の実行で終わらず、フィードバックに基づいて改善する仕組みがあるか
  - イテレーションの停止条件が明確か
- [ ] **ステップの依存関係**: あるステップが前のステップの成果に依存している場合、その依存関係が明示されているか

---

## カテゴリ3: MCP連携強化

代表例: sentry-code-review スキル

### チェック項目

#### ❌ 要修正
- [ ] **MCPコールのシーケンス設計**: 呼び出し順序が根拠とともに設計されているか
  - 各ツールに必要なパラメータとその取得元が明示されているか
  - 複数MCPを使う場合、明確なフェーズ分離がされているか
  - MCP呼び出し間のデータ受け渡しが文書化されているか

#### ⚠️ 改善推奨
- [ ] **ドメイン専門知識の埋め込み**: ユーザーが毎回指定しなくても済むようなコンテキストやベストプラクティスが組み込まれているか
  - 例: 「Sentryのエラーデータを分析する際は、まずスタックトレースのパターンを分類し...」
- [ ] **エラーハンドリングの具体性**: MCP障害時の対処が具体的な手順として記載されているか
  1. MCPサーバーの接続状態の確認方法
  2. 認証（APIキー、OAuthトークン）の確認方法
  3. 再接続手順
  4. MCPを使わずにスキルの機能を独立テストする方法
- [ ] **段階的な機能低下の設計**: MCPが利用できない場合でも部分的に機能するよう設計されているか（該当する場合）
  - MCPなしでも提供できる機能の範囲が特定されているか
  - フォールバック時のユーザー体験が考慮されているか
