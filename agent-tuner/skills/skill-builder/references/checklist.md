# スキル品質チェックリスト

スキルの作成・レビュー時に使用する品質チェックリストです。

## 構造チェック

- [ ] フォルダ名がkebab-caseである
- [ ] SKILL.md が正確な大文字小文字で存在する（SKILL.MD や skill.md は不可）
- [ ] YAMLフロントマターが `---` で正しく囲まれている
- [ ] nameフィールドがkebab-caseである（スペース・大文字・アンダースコアなし）
- [ ] nameフィールドがフォルダ名と一致している
- [ ] descriptionフィールドが1024文字以内である
- [ ] フロントマターにXMLタグ（< >）が含まれていない
- [ ] 名前に「claude」「anthropic」が含まれていない
- [ ] スキルフォルダ内にREADME.mdが存在しない

## description品質チェック

- [ ] 「何をするか」が明記されている
- [ ] 「いつ使うか」（トリガー条件）が明記されている
- [ ] ユーザーが実際に使いそうなフレーズが含まれている
- [ ] 曖昧すぎない（「プロジェクトを手伝う」のような表現は不可）
- [ ] 広すぎない（過剰トリガーのリスクがない）
- [ ] 関連するファイル形式があれば言及されている

## 指示の品質チェック

- [ ] 指示が具体的で実行可能である（「適切に処理する」ではなく具体的な手順）
- [ ] ステップが明確に番号付けまたは見出し付けされている
- [ ] 各ステップの期待結果が明記されている
- [ ] エラーハンドリングが含まれている
- [ ] 使用例が少なくとも1つ含まれている
- [ ] 重要な指示がSKILL.mdの上部に配置されている

## 段階的開示チェック

- [ ] SKILL.mdが5,000語以内に収まっている
- [ ] 詳細なリファレンスがreferences/に分離されている
- [ ] 参照先のファイルパスが正しく記述されている（例：`references/api-patterns.md` を参照）
- [ ] インラインで書くべきでない詳細情報がSKILL.mdに残っていない

## スクリプト・アセット活用チェック

- [ ] 確定的な処理（バリデーション、データ変換など）にスクリプトが活用されているか検討済み
- [ ] スクリプトがある場合、scripts/ディレクトリに配置されている
- [ ] テンプレートやフォントなどの静的リソースはassets/に配置されている
- [ ] スクリプトの実行コマンドがSKILL.mdに明記されている
- [ ] スクリプトの期待出力が明記されている

## MCP連携チェック（該当する場合のみ）

- [ ] 使用するMCPツール名が正確に記述されている
- [ ] MCPツールのパラメータが明示されている
- [ ] MCPサーバー未接続時のエラーハンドリングがある
- [ ] 認証エラーへの対処法が記載されている

## テスト観点

### トリガーテスト
- [ ] 明らかに該当するリクエストでトリガーされるか
- [ ] 言い換えたリクエストでもトリガーされるか
- [ ] 無関係なリクエストではトリガーされないか

デバッグ方法：Claudeに「[スキル名]スキルはいつ使いますか？」と聞く。Claudeがdescriptionを引用するので、不足している内容を確認できる。

### 機能テスト
- [ ] 正常系のワークフローが完了するか
- [ ] APIコール（MCP含む）が成功するか
- [ ] エラー時に適切にハンドリングされるか
- [ ] エッジケースが考慮されているか

### パフォーマンス比較（定量目標の目安）
- [ ] スキルなしと比較して、やり取りの往復が減少するか
- [ ] スキルなしと比較して、失敗するAPIコールが減少するか
- [ ] ユーザーが追加の指示なしでワークフローを完了できるか
- [ ] 複数回実行して一貫した結果が得られるか
- [ ] 他のスキルと同時に有効にしても問題ないか

注：これらは厳密な閾値ではなく、おおよその目安として使う。

## 配布前チェック（GitHub等で公開する場合）

- [ ] リポジトリのルートにREADME（人間向け）を配置している
- [ ] スキルフォルダ内にはREADME.mdを含めていない
- [ ] インストール手順が記載されている
- [ ] 使用例とスクリーンショットがある
- [ ] MCP連携スキルの場合、MCPドキュメントからスキルへのリンクがある
