# MAGI - 設計合意形成システム

3つの異なる設計哲学を持つエージェントが並列で設計案を作成し、合意形成を経て統合された最終設計案を提示するシステムです。

## コマンド

- `/magi`
  - 3つのサブエージェントを並列で起動し、それぞれの設計案を収集・分析・統合して最終案を提示します。
  - エージェント間の直接対話はなく、メインエージェントが各案を分析し統合を行います。

- `/magi-team`
  - `/magi` のエージェントチーム機能を活用したバージョンです。
  - 3つのエージェントが案を作成後に直接議論し、互いの設計案を評価・批判・改善提案をして合意形成を行います。
  - メインエージェントはファシリテーターとして必要に応じて議論の調整や最終報告を行います。

> `/magi-team` の利用には事前設定が必要です。詳しくは[セットアップ](#magi-team-のセットアップ)を参照してください。

## サブエージェント

- `Pragmatic Engineer`
  - シンプルさと保守性を最優先。YAGNI 原則に基づき、実績のある技術と段階的改善を重視する
- `Scalability Architect`
  - 将来の拡張性とパフォーマンスを最優先。疎結合アーキテクチャとクラウドネイティブなアプローチを重視する
- `DX Advocate`
  - 開発者体験を最優先。型安全性・自己文書化・開発効率の向上と一貫性のある設計を重視する

## `/magi-team` のセットアップ

`/magi-team` はエージェントチーム機能（実験的機能）を使用します。事前に以下の設定で機能を有効化してください。

`~/.claude/settings.json` に以下を追加します：

```json
{
  "env": {
    "CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS": "1"
  }
}
```

設定後、Claude Code を再起動すると `/magi-team` が利用可能になります。
