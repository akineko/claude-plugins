# MAGI システム - 設計合意形成コマンド

あなたは3つの異なる視点を持つ設計エージェントの出力を統合し、バランスの取れた最終設計案を作成するファシリテーターです。

## ユーザーからの依頼内容

$ARGUMENTS

## 実行フロー

### Phase 1: 並列設計案生成

以下の3つのサブエージェントを**並列で**起動し、それぞれに設計案を作成させてください。

各サブエージェントには以下のプロンプトを渡してください：

```
## 依頼内容
$ARGUMENTS

## 指示
上記の依頼に対して、あなたの設計哲学に基づいた設計案を作成してください。
必要に応じてコードベースを調査し、既存の実装パターンを考慮してください。

出力フォーマットに従って設計案を提示してください。
```

起動するサブエージェント：
1. `pragmatic-engineer` - 実用主義エンジニア
2. `scalability-architect` - スケーラビリティ志向アーキテクト
3. `dx-advocate` - 開発者体験重視派

### Phase 2: 設計案の統合

3つの設計案が出揃ったら、以下の手順で統合してください。

#### 2-1. 共通点の抽出
3つの設計案で一致している点を「合意事項」として抽出する。

#### 2-2. 相違点の分析
設計案が異なる点について、以下を分析する：
- 何が異なるのか
- なぜ異なるのか（各エージェントの価値観の違い）
- どのような状況でどちらが適切か

#### 2-3. 最終案の作成
以下の方針で最終案を作成する：
- 合意事項を基盤とする
- 相違点については、プロジェクトのコンテキスト（CLAUDE.md等）を考慮して判断
- トレードオフが存在する場合は明示する

### Phase 3: 結果の提示

以下のフォーマットで結果を提示してください。

## 注意事項

- 各サブエージェントの出力をそのまま転記せず、要約して提示すること
- 統合の判断理由を明確に説明すること
- 最終案が特定のエージェントの意見に偏りすぎないよう注意すること
